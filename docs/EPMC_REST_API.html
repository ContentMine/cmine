Web Service Overview

Our RESTful Web Service gives you access to all of the publications and related information we hold in the Europe PMC database, which includes the following:

  • Over 31 million publications from various sources, including PubMed, AGRICOLA, and the European Patent Office.
  • We provide the full text article free for more than 3.7 million of these publications, of which more than 1.2 million are open access.
  • Database cross-references to a number of databases, including UniProt, the European Nucleotide Archive (ENA; formerly EMBL), the Protein Data Bank in Europe (PDBe), and more.
  • Reference lists for more than 5 million publications.
  • Citation counts and a citation network.
  • Terms that have been text-mined from full text articles, including accession numbers, chemicals, diseases, genes and proteins, Gene Ontology terms, and organisms.
  • Users can also read the SOAP Web Service Reference Guide for background information on Europe PMC content (see Sections 2 and 3). See also: What am I searching on Europe PMC?
  • Users may wish to apply search fields to retrieve data, for example when using the ‘search’ module. View the ‘search’ module ‘query’ parameter section below for examples. Also see the tips at the end of each module section. A comprehensive list of search terms can be found on the Help page, and also in the SOAP Web Service Reference Guide.
  • The output response formats can be XML or JSON. In the case of the ‘search’ module a Dublin Core format is also available.
  • Two versions of the RESTful web service are simultaneously available. This approach to release management allows users to prepare for a new version, rather than having to immediately respond to a version change. The details of web service releases will be communicated to all known users. A mailing list is compiled from users that have supplied an address in the 'email' parameters of the 'search' and 'profile' modules. See the 'URL syntax' section below for details on how different versions of the web service can be accessed.

Sorting Results

By default, the Europe PMC RESTful search results are sorted by relevance, with the most relevant result being presented first in the list. RESTful search results can be sorted in two other ways using the arguments: ‘sort_date:y’ and ‘sort_cited:y’ in the ‘query’ parameter of the modules provided. For example:

Sort by the number of citations, most cited being the first result presented:

http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria%20sort_cited:y

Sort by date of publication, the most recently published article being the first result given:

http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria%20sort_date:y

search

Use this module to search our publication database. For more information on query syntax, search field combination and other curiosities, see the Europe PMC help page (opens in new window). This module adheres to the CORS standard of headers to enable javascript cross-domain search.

URL syntax

Construct your URL for search requests as follows:

GET http://www.ebi.ac.uk/europepmc/webservices/rest/search?query={}[&parameters]

Note that for this module, the user may specify either a slash symbol (/) or a question mark symbol (?) after the method name, and before the query. For example:

search?query={}[&parameters] or search/query={}[&parameters]

Request Parameters

  • query
    This is the only obligatory parameter for the search module.

    Enter your search terms with this parameter. You can use a keyword or combination of keywords (%20 here represents a space):

    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria
    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=human malaria

    Or a phrase with enclosing speech marks:

    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query="human malaria"

    Or a fielded search; for a list of available search fields, see Appendix 1 in our Reference Guide (downloads PDF):

    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=auth:stoehr
    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=journal:pnas

    Or a publication ID; for more information on our data sources, see our Reference Guide (downloads PDF):

    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=ext_id:781840 src:med
    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=ext_id:IND43783977 src:agr
  • resulttype
    The result type can either be idlist, core or lite:
    • idlist returns a list of IDs and sources for the given search terms
    • lite returns key metadata for the given search terms; this is the default value if the parameter is unspecified.
    • core returns full metadata for a given publication ID; including abstract, full text links, and MeSH terms.

    This parameter determines the fields returned by XML and JSON formats, but it has no effect on the DC format.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria&resulttype=idlist
  • page
    Specify the results page you wish to retrieve, where applicable. Page numbers start at 1; default value is 1 (if the parameter is unspecified). Note: the page size can also be set with the pageSize parameter, see next.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria&page=10
  • pageSize
    Specify the number of articles you wish to retrieve in each page. The default value is 25 when the parameter is not used. The maximum allowable number of results per page is 1000.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria&page=2&pageSize=3
  • format
    The format can either be XML, JSON or DC (Dublin Core); the default value is XML if the parameter is unspecified. XML returns the same response as the SOAP web service; see the SOAP Web Service Reference Guide.

    DC returns the following Dublin Core metadata fields, within an RDF/XML wrapper: dc:contributor, dc:creator, dc:date, dc:description, dc:identifier, dc:language, dc:subject, dc:title, dc:type, dcterms:abstract, dcterms:accessRights, dcterms:bibliographicCitation, dcterms:isPartOf, dcterms:isVersionOf.

    The use of these fields complies with the DCMI Metadata Terms recommendation documentation. Additionally, it should be noted that dc:contributor fields are used for the organisations to which some, or all of the authors listed in the dc:creator fields are affiliated. dc:description contains the publication types of the record (e.g. "Journal Article", "Book", etc.) dcterms:accessRights is only included for works in the Open Access subset of the PMC collection. There are two dcterms:bibliographicCitation fields for each journal article record, one which matches the "Text (citation)" option in the export feature on the Europe PMC web site, the other a machine-readable format using the syntax of the OpenURL standard. Other publication types (e.g. books, patents) only contain a single, non-machine-readable dcterms:bibliographicCitation.

    Publication dates have been mapped as follows: dc:date - first publication date (the date of first publication, whichever is first, electronic or print publication; where a date is not fully available e.g. year only, an algorithm is applied to determine the value), dc:terms:available - electronic publication date, dc:terms:created - print publication date.

    Note that the ‘resulttype’ parameter associated with the DC response is always set to ‘core’.

    Example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria&format=xml
    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria&format=json
    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria&amp;format=dc
  • callback
    Use the callback parameter to make cross-domain requests and retrieve JSON data. The value is user specified. The web service wraps its response with the given callback value.

    Format value must be JSON.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria&format=json&callback=hello
  • synonym
    Synonym searches are not made by default (parameter value = ‘false’), however queries can be expanded using MeSH terminology and the UniProt synonyms list. For example aspirin, a synonym would be acetylsalicylic acid;this could be included in the search by setting the parameter value to ‘true’.

    Where specified synonym parameter values must be set to ‘true’ or ‘false’.

    Example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=aspirin&synonym=true
  • email
    This optional parameter registers the user's email address with EBI. The user can then be contacted about Web Service related news.

    Example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=flu&email=abcde@abcde.com

profile

Use this module to obtain a 'profile' of hit counts from our database for various publication types, data sources, and subsets. This module adheres to the CORS standard of headers to enable javascript cross-domain search.

URL syntax

Construct your URL for profile requests as follows:

GET http://www.ebi.ac.uk/europepmc/webservices/rest/profile?query={}[&parameters]

Note that for this module, the user may specify either a slash symbol (/) or a question mark symbol (?) after the method name, and before the query. For example: profile?query={}[&parameters] or profile/query={}[&parameters]

Request Parameters

  • query
    This is the only obligatory parameter for the profile module.

    Enter your search terms with this parameter. You might use a keyword or combination of keywords (%20 here represents a space):

    http://www.ebi.ac.uk/europepmc/webservices/rest/profile?query=malaria

    Or a phrase with enclosing speech marks:

    http://www.ebi.ac.uk/europepmc/webservices/rest/profile?query="human malaria"

    http://www.ebi.ac.uk/europepmc/webservices/rest/profile?query=journal:pnas
  • profiletype
    The profile type can either be pub_type, source, subset or all:
    • pub_type returns a profile of various publication types.

citations

Use this module to retrieve a count and list of publications which cite a given publication.

URL syntax

Construct your URL for citations requests as follows:

GET http://www.ebi.ac.uk/europepmc/webservices/rest/src/ext_id/citations/[page]/[pageSize]/[format]/[callback]

Request Parameters

  • src
    This is an obligatory parameter for the citations module.

    The source (src) is a three letter code representing one of our various data sources:
    • AGR = AGRICOLA publications from the U.S. National Agricultural Library
    • CBA = publications from the Chinese Biological Abstract journal, provided by the Shanghai Institutes for Biological Sciences
    • CTX = CiteXplore publications from the European Bioinformatics Institute
    • ETH = EthOs PhD theses from the British Library
    • HIR = clinical guidelines from the U.K. National Health Service
    • MED = publications from PubMed and MEDLINE, curated by the U.S. National Institutes of Health's National Library of Medicine
    • PAT = Biomedical patents from the European Patent Office and patent offices worldwide
    • PMC = publications from PubMed Central, curated by the U.S. National Institutes of Health's National Library of Medicine

    You can find the src for a publication in the 'source' element returned by the search module. Insert this into your URL along with the publication's external ID, as follows:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/ext_id/citations
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/ext_id/citations

    For more information about our data sources, and the total number of publications we currently hold for each source, see our Reference Guide (downloads PDF).

  • ext_id
    This is an obligatory parameter for the citations module.

    You can find a publication's external ID (ext_id) in the 'pmid' element returned by the search module. Insert this into your URL along with the publication source, like the following example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/9843981/citations
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/PMC193660/citations
  • page
    Specify the section of the citations list you wish to retrieve, where applicable. Page numbers start at 1; default value is 1 (if the parameter is unspecified). Note: the page size can also be set with the pageSize parameter, see next.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/9843981/citations/10
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/PMC193660/citations/2
  • pageSize
    Specify the number of citations you wish to retrieve in each page. The default value is 25 when the parameter is not used. The maximum allowable number of citations per page is 1000.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/9843981/citations/3/2
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/PMC193660/citations/4/10
  • format
    The format can either be XML or JSON; the default value is XML if the parameter is unspecified.

    Note that use of the format and page size parameters is optional, when not used they default to values of 1 and 25 respectively.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/9843981/citations/5/json
  • callback
    Use the callback parameter to make cross-domain requests and retrieve JSON data. The value is user specified. The web service wraps its response with the given callback value.

    Note that use of the format and page size parameters is optional, when not used they default to values of 1 and 25 respectively.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/9843981/citations/3/10/json/hello

Tip!

Add sort_cited:y to your search request to quickly find publications that have been cited: 
http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria%20sort_cited:y

databaseLinks

Use this module to retrieve a count and list of biological database records that have cited a given publication (cross references).

URL syntax

Construct your URL for databaseLinks requests as follows:

GET http://www.ebi.ac.uk/europepmc/webservices/rest/src/ext_id/databaseLinks/[database]/[page]/[pageSize]/[format]/[callback]

Request Parameters

  • src
    This is an obligatory parameter for the databaseLinks module.

    The source (src) is a three letter code representing one of our various data sources:
    • AGR = AGRICOLA publications from the U.S. National Agricultural Library
    • CBA = publications from Chinese Biological Abstracts and the Shanghai Institutes for Biological Sciences
    • CTX = CiteXplore publications from the European Bioinformatics Institute
    • ETH = EthOs PhD theses from the British Library
    • HIR = clinical guidelines from the U.K. National Health Service
    • MED = publications from PubMed and MEDLINE, curated by the U.S. National Institutes of Health's National Library of Medicine
    • PAT = Biomedical patents from the European Patent Office and patent offices worldwide
    • PMC = publications from PubMed Central, curated by the U.S. National Institutes of Health's National Library of Medicine

    You can find the src for a publication in the 'source' element returned by the search module. Insert this into your URL along with the publication's external ID, as follows: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/ext_id/databaseLinks
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/ext_id/databaseLinks
    For more information about our data sources, and the total number of publications we currently hold for each source, see our Reference Guide (downloads PDF).
  • ext_id
    This is an obligatory parameter for the databaseLinks module.

    You can find a publication's external ID (ext_id) in the 'pmid' element returned by the search module. Insert this into your URL along with the publication source, like the following example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/10779411/databaseLinks
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/12368864/databaseLinks
  • database
    You can restrict the response to a specific database from the following list; default is to return links for all databases:
    • ARXPR is a database of functional genomics experiments
    • ChEBI is a database and ontology of chemical entities of biological interest
    • ChEMBL is a database of bioactive drug-like small molecules
    • EMBL (now ENA) provides a comprehensive record of the world's nucleotide sequencing information
    • IntAct provides a freely available, open source database system and analysis tools for molecular interaction data
    • InterPro provides functional analysis of proteins by classifying them into families and predicting domains and important sites
    • OMIM is a comprehensive and authoritative compendium of human genes and genetic phenotypes
    • PDB (or PDBe) is the European resource for the collection, organisation and dissemination of data on biological macromolecular structures
    • UniProt provides a comprehensive and freely accessible resource of protein sequence and functional information

    Specify the database you wish to retrieve links for using the first word for each bullet point above in upper case (i.e. ARXPR, CHEBI, CHEMBL, EMBL, INTACT, INTERPRO, OMIM, PDB, UNIPROT). For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/10779411/databaseLinks/PDB
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/12368864/databaseLinks/UNIPROT
  • page
    Specify the section of the database links list you wish to retrieve, where applicable. Page numbers start at 1; default value is 1 (if the parameter is unspecified). Note: the page size can also be set with the pageSize parameter, see next.

    Note that to use the page parameter you must specify a value for the preceding database parameter, even if left empty.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/12368864/databaseLinks//2
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/12368864/databaseLinks/UNIPROT/3
  • pageSize
    Specify the number of database links you wish to retrieve in each page. The default value is 25 when the parameter is not used. The maximum allowable number of database links per page is 1000.

    Note that to use the pageSize parameter you must specify a value for the database parameter, even if left empty.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/12368864/databaseLinks//3/2
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/12368864/databaseLinks/UNIPROT/3/2
  • format
    The format can either be XML or JSON; the default value is XML if the parameter is unspecified.

    Note that to use the format parameter you must use the preceding database parameter (even if left blank), the page and page size parameters being optional.

    For example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/12368864/databaseLinks//12/json
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/12368864/databaseLinks/UNIPROT/3/xml
  • callback
    Use the callback parameter to make cross-domain requests and retrieve JSON data. The value is user specified. The web service wraps its response with the given callback value.

    Note that to use the callback parameter you must use the preceding database parameter (even if left blank), the page and page size parameters being optional.

    For example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/12368864/databaseLinks//1/json/hello
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/12368864/databaseLinks/UNIPROT/3/5/json/mycallback

Tip!

Add has_xrefs:y to your search request to quickly find publications for which we have external database links: 
http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria%20has_xrefs:y

Or just use query=has_xrefs:y —

http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=has_xrefs:y

Similarly, you can use has_chebi:y, has_chembl:y, has_embl:y, and so on.

labsLinks

This module gives access to the External Links provided by 3rd parties (to extend Europe PMC content). See the Europe PMC External Links Service page for more details. Use this module to return the External Links for a given source (src) and identifier (ext_id) combination, or by the external content provider (providerId).

URL syntax

Construct your URL for labsLinks requests as follows:

GET http://www.ebi.ac.uk/europepmc/webservices/rest/src/ext_id/labsLinks/[providerId[s]]/[page]/[format]/[callback]

Request Parameters

  • src
    This is an obligatory parameter for the labsLinks module.

    The source (src) is a three letter code representing one of our various data sources:
    • AGR = AGRICOLA publications from the U.S. National Agricultural Library
    • CBA = publications from Chinese Biological Abstracts and the Shanghai Institutes for Biological Sciences
    • CTX = CiteXplore publications from the European Bioinformatics Institute
    • ETH = EthOs PhD theses from the British Library
    • HIR = clinical guidelines from the U.K. National Health Service
    • MED = publications from PubMed and MEDLINE, curated by the U.S. National Institutes of Health's National Library of Medicine
    • PAT = Biomedical patents from the European Patent Office and patent offices worldwide
    • PMC = publications from PubMed Central, curated by the U.S. National Institutes of Health's National Library of Medicine

    You can find the src for a publication in the 'source' element returned by the search module. Insert this into your URL along with the publication's external ID, as follows: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/ext_id/labsLinks
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/ext_id/labsLinks
    For more information about our data sources, and the total number of publications we currently hold for each source, see our Reference Guide (downloads PDF).
  • ext_id
    This is an obligatory parameter for the labsLinks module.

    Find the External Links associated with a publication's specified src and ext_id combination. You can find a publication's external ID (ext_id) in the 'pmid' element returned by the search module. Insert this into your URL along with the publication source, like the following example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/24727771/labsLinks
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/PMC3986813/labsLinks
  • providerId[s]
    List the External Links for a specified providerId (nnnn). You can find a publication's external ID (ext_id) in the 'pmid' element returned by the search module. Insert this into your URL along with the publication source, like the following example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/PMC3936813/labsLinks/nnnn
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/PMC3936813/labsLinks/nnnn OR nnnn
  • page
    Specify the page of the external links list you wish to retrieve, where applicable, 10 entities at a time; default value is 1 (i.e. entities 1-10), if the parameter is unspecified.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/24023770/labsLinks//2
  • format
    The format can either be XML or JSON; the default value is XML if the parameter is unspecified.

    Note that to use the format parameter you must specify a value for the page parameter (>0).

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/24727771/labsLinks//1/json
  • callback
    Use the callback parameter to make cross-domain requests and retrieve JSON data. The value is user specified. The web service wraps its response with the given callback value.

    Note that to use the callback parameter you must specify a value for the page parameter (>0), and the format must be specified as JSON.

    For example:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/24727771/labsLinks//1/json/hello

Tip!

Use the search module to list all citations with External Links. 
http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=HAS_LABSLINKS:y

textMinedTerms

Use this module to retrieve a count and list of terms we have text-mined from full text publications.

URL syntax

Construct your URL for textMinedTerms requests as follows:

GET http://www.ebi.ac.uk/europepmc/webservices/rest/src/ext_id/textMinedTerms/[semantic_type]/[page]/[pageSize]/[format]/[callback]

Request Parameters

  • src
    This is an obligatory parameter for the textMinedTerms module.

    The source (src) is a three letter code representing one of our various data sources. Note that we only text-mine full text articles — i.e. from PubMed Central — but that publications from this source also 'map' to a MEDLINE ID. Therefore the only sources relevant to textMinedTerms are MED and PMC:
    • MED = publications from PubMed and MEDLINE, curated by the U.S. National Institutes of Health's National Library of Medicine
    • PMC = publications from PubMed Central, curated by the U.S. National Institutes of Health's National Library of Medicine

    Insert the source into your URL along with the publication's external ID, as follows:

    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/ext_id/textMinedTerms
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/ext_id/textMinedTerms
    For more information about our data sources, and the total number of publications we currently hold for each source, see our Reference Guide (downloads PDF).
  • ext_id
    This is an obligatory parameter for the textMinedTerms module.

    You can find a publication's external ID (ext_id) in the 'pmid' or 'pmcid' elements returned by the search module. Insert this into your URL along with the publication source, like the following example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/271968/textMinedTerms
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/PMC431765/textMinedTerms
  • semantic_type
    You can restrict the response to a semantic type from the following list; default is to return text-mined terms of all semantic types:
    • ACCESSION = accession IDs for DNA or protein sequences
    • CHEMICAL
    • DISEASE
    • EFO = Experimental Factor Ontology
    • GENE_PROTEIN
    • GO_TERM = a Gene Ontology term
    • ORGANISM

    Specify the semantic type you wish to retrieve text-mined terms for in upper case as shown above. For example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/271968/textMinedTerms/CHEMICAL
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/PMC431765/textMinedTerms/ORGANISM
    http://www.ebi.ac.uk/europepmc/webservices/rest/PMC/PMC2234103/textMinedTerms/EFO
  • page
    Specify the section of the text mined terms list you wish to retrieve, where applicable. Page numbers start at 1; default value is 1 (if the parameter is unspecified). Note: the page size can also be set with the pageSize parameter, see next.

    Note that to use the page parameter you must specify a value for the preceding semantic_type parameter, even if left empty.

    For example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/271968/textMinedTerms//2
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/271968/textMinedTerms/CHEMICAL/2
  • pageSize
    Specify the number of text mined terms you wish to retrieve in each page. The default value is 25 when the parameter is not used. The maximum allowable number of text mined terms per page is 1000.

    Note that to use the pageSize parameter you must specify a value for the semantic_type parameter, even if left empty.

    For example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/271968/textMinedTerms//2/3
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/271968/textMinedTerms/CHEMICAL/3/5
  • format
    The format can either be XML or JSON; the default value is XML if the parameter is unspecified.

    Note that to use the format parameter you must use the preceding semantic_type parameter, even if no value is specified. The page and page size parameters are optional.

    For example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/271968/textMinedTerms//2/5/xml
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/271968/textMinedTerms/CHEMICAL/2/json
  • callback
    Use the callback parameter to make cross-domain requests and retrieve JSON data. The value is user specified. The web service wraps its response with the given callback value.

    Note that to use the callback parameter you must also use the semantic_type parameter, even if no value is specified. The page and page size parameters are optional.

    For example: 
    http://www.ebi.ac.uk/europepmc/webservices/rest/MED/271968/textMinedTerms/CHEMICAL/2/json/hello

Tip!

Add has_tm:y to your search request to quickly find publications for which we have text-mined terms: 
http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=malaria%20has_tm:y

Or just use query=has_tm:y —

http://www.ebi.ac.uk/europepmc/webservices/rest/search?query=has_tm:y

Most publications will have both pmid and pmcid elements in the response. Some publications we haven't yet 'mapped' to each other will only show one of these; in this case please keep checking back to find the mapping complete.
